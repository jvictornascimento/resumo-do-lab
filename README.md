# resumo-do-lab
Este repositório contém o resumo das lições aprendidas durante o desenvolvimento do lab na DIO

## Especificações de principal diferença entre CapEX e OpEx.

### CapEx
- O gasto inicial de dinheiro em infraestrutura física.
- As despesas do CapEx têm um valor que se reduz com o tempo.
resumo: tudo que for de maquina física que eu adiquirir


### OpEx
- Gastar com produtos e serviços conforme necessário, pagamento conforme o uso.
- Seja cobrado imediatamente.
resumo: tudo de logico que eu provisionar e instanciar.

## Cloud 

### Pública
- Infraestrutura pertence a um provedor externo
- Ex: AWS, Google Cloud, Microsoft Azure
- Vários clientes compartilham os recursos (multi-tenant)
- Escalável e barato (pay-as-you-go)
- Menos controle direto
resumo: Use quando: quer rapidez, baixo custo e não quer gerenciar servidor

### Privada
- Infraestrutura exclusiva de uma empresa
- Pode ser local (on-premise) ou hospedada
- Mais segurança e controle
- Mais caro e complexo
resumo: Use quando: precisa de alta segurança, compliance ou dados sensíveis

### Híbrida
- Mistura cloud pública + privada
- Permite distribuir cargas (ex: dados sensíveis privado, app público)
- Equilíbrio entre custo e controle
- Mais complexa de gerenciar
resumo: Use quando: precisa flexibilidade e quer migrar aos poucos

## Geral
- Como planejar a melhor opção de cloud.
- Como criar uma conta e utilizar os recursos gratuitos do Azure.
- Entender que tudo que criamos quando acabamos de testar, ja devemos excluir para não haver cobranças surpresas, e caso haja há maneiras de solicitar reembolso mas leva tempo e não é certo.
- Enteder o porque devemos usar o painel de previsão de gastos para ter um controle mais refinado do que vamos ter de gastos mês a mês.
